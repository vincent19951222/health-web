- 最近更新时间：2026-01-08 19:49:13（页面显示）

## 简介

本文档介绍如何通过 WebSocket 协议实时访问大模型流式语音识别服务（ASR），包含鉴权、协议详情、错误码和 Demo。

### 接口地址

- 双向流式：`wss://openspeech.bytedance.com/api/v3/sauc/bigmodel`
- 流式输入（nostream）：`wss://openspeech.bytedance.com/api/v3/sauc/bigmodel_nostream`
- 双向流式优化版：`wss://openspeech.bytedance.com/api/v3/sauc/bigmodel_async`

### 模式说明（页面要点）

1. 双向流式：每输入一包通常返回一包，首字更快。
2. 流式输入（nostream）：音频>15s或最后一包后返回，准确率更高。
3. 建议音频分包大小与发包间隔均为 `100~200ms`（双向流式建议 `200ms`）。
4. nostream 在平均 5s 音频时，返回可到 `300~400ms` 以内。

## 鉴权

WebSocket 建连的 HTTP Header 需包含：

- `X-Api-App-Key`：控制台获取的 APP ID
- `X-Api-Access-Key`：控制台获取的 Access Token
- `X-Api-Resource-Id`：资源 ID（示例）
  - 豆包流式语音识别模型1.0
    - 小时版：`volc.bigasr.sauc.duration`
    - 并发版：`volc.bigasr.sauc.concurrent`
  - 豆包流式语音识别模型2.0
    - 小时版：`volc.seedasr.sauc.duration`
    - 并发版：`volc.seedasr.sauc.concurrent`
- `X-Api-Connect-Id`：建议 UUID，用于链路追踪

握手成功后建议记录响应头：

- `X-Api-Connect-Id`
- `X-Tt-Logid`（排错关键）

### 建连 Header 示例

```http
GET /api/v3/sauc/bigmodel
Host: openspeech.bytedance.com
X-Api-App-Key: 123456789
X-Api-Access-Key: your-access-key
X-Api-Resource-Id: volc.bigasr.sauc.duration
X-Api-Connect-Id: 随机生成的UUID
```

## 协议详情

WebSocket 使用二进制协议，核心由：

1. `header`（至少 4 字节，可扩展）
2. `payload size`（长度）
3. `payload`（负载）

> 整数字段均为大端（Big Endian）。

### Header 字段要点

- `Protocol version`：`0b0001`（当前版本）
- `Header size`：`0b0001`（4 字节）
- `Message type`：
  - `0b0001`：full client request
  - `0b0010`：audio only request
  - `0b1001`：full server response
  - `0b1111`：error response
- `Message type specific flags`：
  - `0b0000`：后 4 字节不是 sequence
  - `0b0001`：后 4 字节是正 sequence
  - `0b0010`：标记最后一包（负包）
  - `0b0011`：后 4 字节为负 sequence（最后一包）
- `Serialization`：`0b0000`（none） / `0b0001`（JSON）
- `Compression`：`0b0000`（none） / `0b0001`（Gzip）

## 请求流程

### 1) 建立连接

按 WebSocket 协议发起 GET 升级，并带鉴权 Header。

### 2) 发送 full client request

连接建立后，第一个包为 full client request，包含音频元数据与请求参数（通常 JSON + Gzip）。

#### 关键参数（节选）

- `audio`
  - `format`：`pcm/wav/ogg/mp3`
  - `codec`：`raw/opus`（ogg 必须 opus）
  - `rate`：当前仅 `16000`
  - `bits`：当前仅 `16`
  - `channel`：`1/2`
  - `language`：仅 `bigmodel_nostream` 支持指定语种
- `request`
  - `model_name`：当前 `bigmodel`
  - `enable_nonstream`：双向流式优化版支持二遍识别
  - `enable_itn` / `enable_punc` / `enable_ddc`
  - `show_utterances`
  - `show_speech_rate` / `show_volume`
  - `enable_lid`（语种检测）
  - `enable_emotion_detection` / `enable_gender_detection`
  - `result_type`：`full` 或 `single`
  - `enable_accelerate_text` / `accelerate_score`
  - `vad_segment_duration`
  - `end_window_size`（默认 800ms，最小 200）
  - `force_to_speech_time`
  - `sensitive_words_filter`
  - `enable_poi_fc` / `enable_music_fc`
  - `corpus`（热词、替换词、上下文）

#### `language`（文档列出）

- `en-US`, `ja-JP`, `id-ID`, `es-MX`, `pt-BR`, `de-DE`, `fr-FR`, `ko-KR`, `fil-PH`, `ms-MY`, `th-TH`, `ar-SA`

### 3) 发送 audio only request

后续持续发送音频包（按 full client request 指定的音频格式/编码/采样率/声道）。

- 可多次发送，例如每次发送 100ms 音频。
- 最后一包通过 `message type specific flags` 标识。

### 4) 接收 full server response

服务端会对 full client request 与 audio only request 返回 full server response。

返回 payload 为 JSON，核心字段包含：

- `result.text`：整段识别文本
- `result.utterances[]`：分句结果（启用 `show_utterances` 时）
  - `text`
  - `start_time`
  - `end_time`
  - `definite`
  - `words[]`（逐词时间）

## Error message from server

当协议或消息无法处理时，服务端返回错误帧（`message type = 0b1111`）。

结构包含：

- Header
- Error message code（4B, uint32）
- Error message size（4B, uint32）
- Error message（UTF-8）

---

## 示例流程（文档示例摘要）

1. client 发送 full client request（JSON + Gzip）
2. server 返回 full server response（sequence=1）
3. client 发送音频包（audio only）
4. server 返回 full server response（sequence=2）
5. client 发送最后一包音频（flags 标记最后一包）
6. server 返回最终结果（flags 标记最后一包结果，sequence=3）
7. 若异常，可能返回 error response（`message type=0b1111`）

---

## 错误码（页面列出）

- `20000000`：成功
- `45000001`：请求参数无效（缺失、值无效、重复请求等）
- `45000002`：空音频
- `45000081`：等包超时
- `45000151`：音频格式不正确
- `550xxxxx`：服务内部处理错误
- `55000031`：服务器繁忙（服务过载）
